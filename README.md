# 合唱ステージ配置メーカー 公開版 v0.17

不特定多数の利用を想定し、運営側の Google スプレッドシートに個人情報や保存済み配置を残さない形へ変更した公開版です。

## v0.17 の主な変更

- `input_members`、`name_entry`、`layouts` などの共有スプレッドシート保存を使わない構成に変更しました。
- 名簿は Web 画面へ貼り付け、または CSV/TSV ファイル読み込みで扱います。
- 保存済み配置は利用者のブラウザ内 `localStorage` に保存します。
- 配置データを `.choruslayout` ファイルとして保存・読み込みできます。
- 名前を後から入れる場合は、名前入力CSVを保存し、利用者側で編集してからCSVを読み込んで反映します。
- GAS側の `Code.gs` はWebアプリ表示だけを行う最小構成です。スプレッドシート権限は不要です。

## ファイル

- `Code.gs`
- `Index.html`
- `appsscript.json`
- `README.md`

Apps Script に `Code.gs`、`Index.html`、`appsscript.json` を配置してWebアプリとしてデプロイしてください。

## データの扱い

公開版では、名簿・配置・名前入力データは運営側のスプレッドシートには保存されません。

保存先は次のいずれかです。

- 利用者のブラウザ内保存: 同じ端末・同じブラウザで使える保存済み配置
- 配置ファイル: `.choruslayout` として利用者が保存・共有できるファイル
- CSV: 名簿や名前入力用の利用者側ファイル

注意: ブラウザ内保存は、ブラウザデータを削除すると消えます。大事な配置は `.choruslayout` ファイルとして保存してください。

## 使い方

### 名簿が既にある場合

1. ExcelやGoogleスプレッドシートで「氏名」「パート」の列をコピーします。
2. 初期画面の名簿貼り付け欄に貼り付けます。
3. 「貼り付け名簿から新規配置を作成」を押します。

CSV/TSVファイルを読み込むこともできます。

推奨形式:

```text
氏名,パート
山田花子,S
佐藤明子,A
鈴木一郎,T
```

またはタブ区切り:

```text
氏名	パート
山田花子	S
佐藤明子	A
```

### 各パートの人数だけ分かっている場合

1. パート構成を選びます。
2. 各パートの人数をWeb画面上で入力します。
3. 「この人数で配置案を作成」を押します。
4. 配置を調整します。
5. 名前を後から入れる場合は「名前入力CSVを保存」を押します。
6. CSVの「氏名」列に名前を入力します。
7. Web画面に戻り「名前CSVを反映」を押します。

### 保存

- 「保存」: このブラウザ内に保存します。
- 「配置ファイル保存」: `.choruslayout` ファイルとして保存します。
- 「配置ファイルを開く」: 保存済み `.choruslayout` またはJSONファイルを読み込みます。

### 出力

- 印刷 / PDF
- PNG画像
- 配置ファイル `.choruslayout`

## 主な機能

- 可変パート対応: SATB、8声部、SSAA、TTBB、カスタム
- 2列〜6列の候補生成
- 配置テンプレート: パート別・縦縞、女声前列/男声後列、男声中央/女声両端、混合、カスタム
- 千鳥配置
- PC: ドラッグ差し込み、範囲選択、ダブルクリック編集
- スマホ: ピンチズーム、通常ドラッグでキャンバス移動、長押しドラッグでカード移動、ダブルタップ編集
- 未配置トレイ
- 元に戻す / やり直し
- PNG画像保存
- 自動下書き保存・復元

## 公開利用時の注意

この公開版は、運営側に利用者の名簿や配置を保存しない設計です。ただし、利用者のブラウザ内保存やダウンロードしたファイルには氏名等の情報が含まれる可能性があります。共有・配布時は利用者自身でファイル管理に注意してください。
